# Информация о проекте Do Good

## Описание

Do Good - это веб-платформа для обмена товарами и добрых дел. Пользователи могут размещать объявления о вещах, которые они готовы отдать бесплатно, и находить нужные им вещи на карте.

## Основные функции

- 📝 Регистрация и авторизация пользователей
- 📍 Размещение объявлений с геолокацией
- 🗺️ Интерактивная карта с объявлениями (Yandex Maps)
- 👤 Личный кабинет пользователя
- ✏️ Редактирование своих объявлений
- 🔍 Поиск и фильтрация объявлений
- 📱 Адаптивный дизайн

## Технический стек

### Frontend
- **React 17** - библиотека для создания пользовательских интерфейсов
- **Redux** - управление состоянием приложения
  - Redux Saga - для асинхронных операций
  - Redux Thunk - альтернативный middleware для async операций
- **React Router v6** - маршрутизация
- **Yandex Maps API** - интеграция карт
- **Bootstrap 5 / Reactstrap** - UI компоненты и стили
- **Axios** - HTTP клиент для API запросов

### Backend
- **Node.js** - серверная платформа
- **Express** - веб-фреймворк
- **PostgreSQL** - реляционная база данных
- **Sequelize** - ORM для работы с БД
- **Express Session** - управление сессиями
- **Multer** - загрузка файлов
- **bcrypt (sha256)** - хеширование паролей

## Структура базы данных

Основные таблицы:
- **Users** - пользователи системы
- **Items** - объявления о товарах
- **Categories** - категории товаров (если есть)

## API Endpoints

### Пользователи (`/users`)
- `POST /users/register` - регистрация
- `POST /users/login` - авторизация
- `GET /users/logout` - выход
- `GET /users/profile` - получение профиля
- `PUT /users/profile` - обновление профиля

### Объявления (`/items`)
- `GET /items` - получение всех объявлений
- `GET /items/:id` - получение одного объявления
- `POST /items` - создание объявления
- `PUT /items/:id` - обновление объявления
- `DELETE /items/:id` - удаление объявления

## Переменные окружения

```env
PORT=3001                    # Порт сервера

DB_PASSWORD=password        # Пароль БД
DB_NAME=dogood_db          # Имя БД
DB_HOST=localhost          # Хост БД
DB_PORT=5432               # Порт БД
COOKIE_SECRET=secret       # Секрет для сессий
COOKIE_NAME=dogood_session # Имя cookie
```

## Разработка

### Запуск в режиме разработки

```bash
# Терминал 1 - Сервер
cd server
npm start

# Терминал 2 - Клиент
cd client
npm start
```

### Работа с базой данных

```bash
# Создание новой миграции
cd server
npx sequelize-cli migration:generate --name add-new-field

# Применение миграций
npx sequelize-cli db:migrate

# Откат последней миграции
npx sequelize-cli db:migrate:undo

# Создание сида
npx sequelize-cli seed:generate --name demo-users

# Применение сидов
npx sequelize-cli db:seed:all
```

## Деплой

### Подготовка к продакшену

1. Создайте production базу данных
2. Настройте переменные окружения для production
3. Соберите клиент: `cd client && npm run build`
4. Настройте сервер для раздачи статики из `client/build`

### Рекомендации
- Используйте HTTPS в продакшене
- Настройте CORS правильно
- Используйте переменные окружения для всех секретов
- Настройте логирование
- Добавьте мониторинг
